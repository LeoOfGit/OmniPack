1. 界面布局全解析

OmniPack 的主界面经过精心设计，分为四个核心交互区域。

1.1 顶层工具栏 (Global Toolbar)

这是执行全局搜索、过滤与批量指令的指挥中心。

  • 选择控制 (All/None)：快速全选或全取消当前主界面中所有可见的包。
  • 搜索框 (Search)：实时过滤各个环境卡片中显示的包名，支持模糊匹配。
  • 待更新过滤 (Only Show Outdated)：一键隐藏所有已是最新版本的包，让您可以集中处理需要升级的项目。
  • 全局刷新 (Refresh All)：重新扫描所有配置环境的本地包列表，并查询云端最新版本。
  • 批量更新 (Batch Update)：对当前所有【已勾选】且【云端有新版本】的包发起并行升级指令。
  • 批量卸载 (Batch Remove)：对所有当前勾选的包发起卸载指令（会有二次确认弹窗）。
  • 设置 (Settings)：直接跳转至环境配置与系统设置页面。

1.2 环境主体卡片 (Env Canvas)

左侧区域，以卡片形式展示每一个 Python 虚拟环境或 Node.js 项目。

  • 环境信息：每张卡片顶部显示环境的别名、物理路径和 Python/Node 版本。
  • 操作按钮：卡片内部集成“添加包”、“局部刷新”和“局部更新全部”等快捷入口。
  • 包列表：列出当前环境下安装的所有包及其版本。

1.3 实时控制台 (Live Console)

右侧区域，实时流式输出底层包管理器的执行详情。

  • 分色输出：系统消息（蓝色）、标准输出（白色）、标准错误（橙色）、失败报告（红色）。
  • 右键功能：支持“清空控制台”、“复制选区”以及“打开日志文件所在目录”。

1.4 状态栏反馈 (Status Bar)

位于窗口最底部，提供实时的统计与全局功能入口。

  • 左侧统计区：Envs: {n} | Packages: {m} ( {u} Outdated )。实时反映当前扫描出的环境总数、包总数及待更新统计。
  • 中间状态区：显示当前正在执行的操作（如 "Ready", "Scanning...", "Installing..."）。
  • 右侧切换区：
    • Python / Node.js 按钮：用于在两种包管理面板间快速切换。
    • 💡 Guide 按钮：点击即可弹出本用户指南窗口。
注意：若程序获得了管理员权限，(Admin) 标识会显示在窗口的最上方标题栏。

2. 环境管理

2.1 统一导入流程 (Unified Intake)

在 Settings -> Python/NPM Environments 中，您可以管理项目的生命周期。

  • Python 环境 (Pip/uv)：
    • Detect System：扫描 PATH、注册表及桌面/用户目录下的 Python 解释器。
    • Add Manually：支持指向具体的 python.exe 解释器，或直接拖入包含 .venv 或 venv 的项目根目录。
  • Node.js 环境 (NPM)：
    • Add Manually：支持指向 package.json 文件或其所在的任意父目录。OmniPack 会自动溯源至项目根节点并解析项目名。
  • Batch Paste (批量导入)：您可以用Everything等搜索工具直接复制得到包含多个路径的多行文本，然后粘贴到输入框中，内置的 env_detector 引擎会自动执行去重、有效性验证和智能命名。。

2.2 环境操作说明

  • 拖拽重排：在列表项上长按图标即可上下拖动，调整环境在主界面的首选顺序。
  • 编辑环境：双击任意项可修改显示别名或修正因移动文件夹导致的路径变化。

3. Python (uv/pip) 深度管理

3.1 极速 uv 引擎

OmniPack 原生集成了一流的 uv 引擎。

  • 性能优势：对于复杂的依赖解析和大规模包安装，由于 uv 采用 Rust 编写且具备高效缓存，其速度通常是 pip 的数十倍。
  • 引擎更新：在 Settings -> Backend 设置中，可以一键触发 Check for Update 异步检查并在线更新 uv 引擎，无需重装程序。

3.2 依赖查询与依赖树

  • 层级展开：点击包名前的箭头，可无限向下展示该包引入的子依赖（Dependency Tree）。
  • 顶层视图：默认仅显示您主动安装的包，隐藏由于级联关系引入的被动安装包，极大减少视觉噪音。

3.3 本地缓存驱动的包搜索

  • 秒开响应：OmniPack 内置了超过 50 万笔热门 PyPI 包的数据缓存。在主界面或“添加包”对话框搜索时，其响应速度堪比本地文件搜索。
  • 搜索增强：搜索时支持对结果按版本范围进行筛选。

4. Node.js (npm) 项目管理

4.1 项目深度溯源

当您向 OmniPack 提供一个含有 Node.js 模块的任意深层文件夹时，它能自动确定真正的项目根目录，并精准展示出本地安装的模块版本。

4.2 在线搜索npm包

  • 在线搜索：从npm源搜索包，支持模糊匹配。

4.3 Dist-Tags 精准分发

OmniPack 将 NPM 的 Dist-Tags 机制进行了 GUI 封装。

  • 标签切换：在添加或更新包时，界面会整齐排列出所有远程 Release 通道（如 latest, beta, next, lts）。
  • 一键更新：点击对应标签卡片，系统会自动构造出如 npm install lodash@beta 这样的精准命令并执行。

5. 设置中心 (Settings Center)

设置对话框是 OmniPack 的后台核心，按功能划分为五个清晰的选项卡：

  1. Python/NPM Environments：环境列表维护区，支持增删改查及排序。
  2. Sources (注册表/镜像源)：
    • Follow System：尊重系统原有的配置。
    • Official：使用官方源。
    • Custom Preset：内置清华、阿里、华为等顶级镜像，一键切换。
    • 独立性：此处的设置仅针对 OmniPack 调用的命令生效，不会修改您的系统全局配置。
  1. Backend (核心引擎与缓存)：
    • uv 路径管理：手动指定或自动发现 uv 解释器。
    • PyPI Cache 同步：提供可视化的同步日志流。支持在程序运行期间静默同步几千万条记录，并可以随时随时暂停或取消。
  1. Proxy (代理配置)：
    • 协议分离：支持独立的 HTTP 和 HTTPS 代理设置。后台命令多数情况会使用https，建议两个都填。
    • 颗粒度控制：可单独勾选哪些域名（PyPI / NPM / GitHub）需要通过代理服务器。
    • 连通性测试：一键执行 Direct vs Proxy 响应延迟对比。

7. 常见问题解答 (FAQ)

Q: 某些npm应用已经发布新版本，但程序未能检测到
A: 很多npm的镜像源对nightly、preview等非正式版本的更新不及时，换用官方源即可。

Q: 为什么“状态栏”中的统计数量和卡片中看到的不一致？
A: 状态栏统计的是背景线程中所有已识别环境的总和，而卡片中显示的可能是经过“搜索框”过滤后的结果。

Q: 如何让我的 Python 环境显示在 Settings 列表最前面？
A: 在 Settings 对话框中，点击环境项左侧的拖拽手柄，将其移动到第一行并保存即可。

Q: “控制台”显示更新失败，但包版本却变了？
A: 这可能是由于底层工具（如 npm）在某些网络环境下报告了非零返回码。您可以点击对应卡片的“刷新”按钮查看环境的真实最新状态。

Q: (Admin) 状态是什么触发的？
A: 当 OmniPack 检测到您正在安装包的环境位于系统保护目录（如 C:\Program Files）时，它会自动请求管理员提权。此时标题栏和状态栏均会显示 (Admin) 标记。

OmniPack V4 User Guide

1. Interface Layout Overview

The OmniPack main interface is meticulously designed and divided into four core interactive areas.

1.1 Global Toolbar

This is the command center for performing global searches, filtering, and batch operations.

  • Selection Control (All/None): Quickly select or deselect all visible packages in the current main interface.
  • Search Box: Real-time filtering of package names displayed in each environment card, supporting fuzzy matching.
  • Filter Outdated: Hide all up-to-date packages with one click, allowing you to focus on projects that need upgrading.
  • Refresh All: Rescan local package lists for all configured environments and query the latest versions online.
  • Batch Update: Initiate parallel upgrade commands for all currently [checked] packages that have newer versions available online.
  • Batch Remove: Initiate uninstallation commands for all currently checked packages (a confirmation dialog will appear).
  • Settings: Jump directly to environment configuration and system settings pages.

1.2 Environment Canvas (Env Canvas)

The left area displays each Python virtual environment or Node.js project as a card.

  • Environment Info: The top of each card shows the environment alias, physical path, and Python/Node version.
  • Action Buttons: Cards integrate shortcuts such as "Add Package," "Local Refresh," and "Local Update All."
  • Package List: Lists all packages installed in the current environment along with their versions.

1.3 Live Console

The right area provides real-time streaming output of execution details from underlying package managers.

  • Color-coded Output: System messages (Blue), Standard output (White), Standard error (Orange), Failure reports (Red).
  • Right-click context menu: Supports "Clear Console," "Copy Selection," and "Open Log Directory."

1.4 Status Bar Feedback

Located at the very bottom of the window, providing real-time statistics and global feature entries.

  • Left Statistics Area: Envs: {n} | Packages: {m} ( {u} Outdated ). Reflects the total number of scanned environments, total packages, and pending updates in real-time.
  • Middle Status Area: Displays the current operation being executed (e.g., "Ready", "Scanning...", "Installing...").
  • Right Switching Area:
    • Python / Node.js Buttons: Used for quickly switching between the two package management panels.
    • 💡 Guide Button: Click to pop up this User Guide window.
Note: If the program has obtained administrator privileges, the (Admin) indicator will be displayed in the top title bar of the window.

2. Environment Management

2.1 Unified Intake Process

In Settings -> Python/NPM Environments, you can manage the lifecycle of your projects.

  • Python Environments (Pip/uv):
    • Detect System: Scans PATH, registry, and desktop/user directories for Python interpreters.
    • Add Manually: Supports pointing to a specific python.exe interpreter or directly dragging in a project root directory containing .venv or venv.
  • Node.js Environments (NPM):
    • Add Manually: Supports pointing to a package.json file or any of its parent directories. OmniPack will automatically trace back to the project root and parse the project name.
  • Batch Paste: You can copy multi-line text containing multiple paths using tools like Everything and paste it into the input box. The built-in env_detector engine will automatically perform deduplication, validity verification, and intelligent naming.

2.2 Environment Operations

  • Drag to Reorder: Long-press the icon on a list item to drag it up or down, adjusting the preferred order of environments on the main interface.
  • Edit Environment: Double-click any item to modify the display alias or correct path changes caused by moving folders.

3. Python (uv/pip) Advanced Management

3.1 High-Speed uv Engine

OmniPack natively integrates the top-tier uv engine.

  • Performance Advantage: For complex dependency resolution and large-scale package installations, uv is typically dozens of times faster than pip because it is written in Rust and features efficient caching.
  • Engine Updates: In Settings -> Backend, you can trigger a Check for Update with one click to asynchronously check and update the uv engine online without reinstalling the application.

3.2 Dependency Query and Tree

  • Hierarchical Expansion: Click the arrow before a package name to infinitely expand and display the sub-dependencies introduced by that package (Dependency Tree).
  • Top-level View: By default, only packages you explicitly installed are shown, while passively installed packages from cascading dependencies are hidden, significantly reducing visual noise.

3.3 Local Cache-Driven Package Search

  • Instant Response: OmniPack features built-in data caching for over 500,000 popular PyPI packages. When searching on the main interface or in the "Add Package" dialog, response speeds are comparable to local file searches.
  • Enhanced Search: Search results support filtering by version range.

4. Node.js (npm) Project Management

4.1 Deep Project Tracing

When you provide OmniPack with any deep folder containing Node.js modules, it automatically determines the actual project root and accurately displays the locally installed module versions.

4.2 Online Search for NPM Packages

  • Online Search: Search for packages from the NPM registry with support for fuzzy matching.

4.3 Precise Distribution via Dist-Tags

OmniPack provides a GUI wrapper for NPM's Dist-Tags mechanism.

  • Tag Switching: When adding or updating packages, the interface neatly displays all remote release channels (e.g., latest, beta, next, lts).
  • One-click Update: Click a tag card and the system automatically constructs and executes precise commands like npm install lodash@beta.

5. Settings Center

The Settings dialog is the backend core of OmniPack, divided into five clear tabs by function:

  1. Python/NPM Environments: Environment list maintenance area, supporting CRUD operations and reordering.
  2. Sources (Registries/Mirrors):
    • Follow System: Respect existing system configurations.
    • Official: Use official registries.
    • Custom Preset: Built-in top-tier mirrors like Tsinghua, Alibaba, and Huawei for one-click switching.
    • Independence: Settings here only apply to commands invoked by OmniPack and will not modify your global system configuration.
  1. Backend (Core Engine & Cache):
    • uv Path Management: Manually specify or automatically discover the uv interpreter.
    • PyPI Cache Sync: Provides a visual sync log stream. Supports silent synchronization of tens of millions of records while the program is running, with the ability to pause or cancel at any time.
  1. Proxy (Proxy Configuration):
    • Protocol Separation: Supports independent HTTP and HTTPS proxy settings. Since backend commands mostly use HTTPS, it is recommended to fill both.
    • Granular Control: Individually select which domains (PyPI / NPM / GitHub) should go through the proxy server.
    • Connectivity Test: One-click execution of Direct vs. Proxy response latency comparison.

7. Frequently Asked Questions (FAQ)

Q: Some NPM packages have new versions released, but the program fails to detect them.
A: Many NPM mirrors are not updated in time for non-official versions like nightly or preview; switching to the official registry should resolve this.

Q: Why is the count in the "Status Bar" different from what I see in the cards?
A: The status bar counts the total of all identified environments in the background thread, whereas the cards might show results filtered by the "Search Box."

Q: How can I make my Python environment appear at the top of the Settings list?
A: In the Settings dialog, click and hold the drag handle on the left side of the environment item, move it to the first row, and save.

Q: The "Console" shows an update failure, but the package version has changed?
A: This might happen if the underlying tool (like NPM) reports a non-zero exit code in certain network environments. You can click the "Refresh" button on the corresponding card to see the actual updated status of the environment.

Q: What triggers the (Admin) status?
A: When OmniPack detects that the environment where you are installing packages is located in a system-protected directory (e.g., C:\Program Files), it automatically requests administrator elevation. At this point, both the title bar and status bar will display the (Admin) marker.
